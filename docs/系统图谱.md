# 系统图谱

本文档以文字方式描述当前系统结构，避免在仓库中保留大量生成图片。

## 总体架构

```text
Vue 前端 / 设备 HTTP / MQTT 网关 / 运营商 Webhook
        |
        v
Flask 接入层：认证、权限、API、WebSocket
        |
        v
应用编排层：后台服务运行态、实时广播、跨模块用例
        |
        v

        |
        v

```

## 代码分层关系

```text
src/app.py
  -> src/application/runtime.py
  -> src/application/realtime.py
  -> src/domain/*.py
  -> src/services.py / src/device_integration.py
  -> src/database.py / src/model_registry.py / src/*_gateway.py
```

路由层不直接管理模拟器、训练器、MQTT 网关或漂移监控实例；这些长生命周期对象由应用编排层统一持有。实时推送也通过应用编排层统一触发，确保设备、告警、工单、报表和设置页刷新口径一致。

## 异常检测流程

```text
设备上报单条读数
  -> 写入历史监测表
  -> 查询该设备最近 24 条读数
  -> 若不足 24 条，仅保存数据
  -> 若满足窗口长度，提取 7 个特征
  -> 加载当前 active model 和 scaler
  -> 标准化窗口
  -> 模型重构输入序列
  -> 计算 MSE 重构误差
  -> 与阈值比较得到 predicted_label
  -> 结合低电压、弱信号、流量突变、长时间静止、压力异常等业务规则判断异常类型和等级
  -> 生成告警和可选工单
```

## 持续训练流程

```text
历史数据和人工反馈
  -> 训练数据生成
  -> 数据清洗
  -> 滑动窗口样本构造
  -> 训练候选 LSTM AutoEncoder
  -> 计算验证集重构误差
  -> 选择阈值
  -> 评估 Accuracy / Precision / Recall / F1
  -> 注册模型版本
  -> 指标更优时切换 active model
```

# 测试文档

## 测试目标

验证系统是否能够完成：

- Web 后端启动。
- 数据库连接和表初始化。
- 设备注册与读数上报。
- 24 点滑动窗口构造、7 类特征标准化和 LSTM AutoEncoder 在线检测。
- 低电压、弱信号、流量突变、压力异常等业务规则融合。
- 告警生成、查询和人工反馈。
- 工单创建、派发、接单和完成。
- 持续训练状态查询和触发。
- 虚拟设备、MQTT 网关和数据漂移监控的基础控制。

## 快速健康检查

```powershell
.\validate_project_health.bat
```

该脚本用于检查关键模块是否可导入、应用是否能创建、独立前端脚本语法是否正确，以及关键业务入口是否可冒烟运行。

健康检查脚本默认使用 SQLite，并将 `DRIFT_MONITOR_ENABLED` 设为 `0`，适合作为本地快速回归。它不能替代 MySQL、MQTT、浏览器交互和长时间训练测试。

## 测试环境

| 环境 | 用途 | 推荐配置 |
| --- | --- | --- |
| 本地快速回归 | 验证代码可导入、接口可冒烟、前端脚本语法正确 | `DB_BACKEND=sqlite` |
| 答辩演示 | 验证页面、图表、设备模拟、告警和工单闭环 | 后端 5000、前端 5173、可开启虚拟设备 |
| 准生产验证 | 验证 MySQL、密钥、MQTT、模型文件和部署参数 | `DB_BACKEND=mysql`，固定 `APP_CREDENTIAL_SECRET` |

## 手工测试流程

### 1. Web 服务

```powershell
.\start_system.bat
```

另开终端启动前端：

```powershell
.\start_frontend.bat
```

检查：

- 浏览器访问 `http://127.0.0.1:5173`。
- 登录页、仪表盘、设备页、告警页、工单页能正常打开。
- `/api/dashboard` 返回 JSON。

### 2. 设备注册

请求：

```http
POST /api/device/register
Content-Type: application/json
```

示例请求体：

```json
{
  "meter_id": "GM-TEST-001",
  "name": "测试燃气表",
  "location": "测试小区",
  "protocol": "HTTP"
}
```

检查：

- 返回设备信息和通信密钥。
- 设备列表出现新设备。
- 数据库设备表新增记录。

### 3. 设备上报

请求：

```http
POST /api/device/upload
X-API-Key: <设备密钥>
Content-Type: application/json
```

示例请求体：

```json
{
  "meter_id": "GM-TEST-001",
  "instant_flow": 0.36,
  "cumulative_usage": 128.52,
  "battery_voltage": 3.42,
  "signal_strength": 76,
  "valve_state": 1,
  "temperature": 22.5,
  "pressure": 2.1
}
```

检查：

- 历史数据新增。
- 设备最后在线时间更新。
- 数据量达到 24 条后出现异常得分和预测标签。

### 4. 异常告警

构造低电压、弱信号、流量突增或压力异常数据。

检查：

- 告警表新增记录。
- 告警页面显示异常类型、得分、阈值、严重程度。
- 前端实时刷新或手动刷新后可见。

### 5. 工单闭环

检查：

- 可从告警创建工单。
- 可选择工程师。
- 工程师可接单。
- 工单可标记完成并填写处理结果。
- 处理结果可作为人工反馈样本。

### 6. 持续训练

在“系统设置”页面开启持续训练开关。

检查：

- `/api/training` 能返回训练状态。
- 候选模型训练完成后写入模型元数据。
- 指标优于当前模型时可切换 active model。

可选导出论文实验数据：

```powershell
.\export_lstm_autoencoder_experiment_results.bat
```

检查 `data/experiment_results/` 是否生成方法对比、ROC、阈值扫描、重构曲线和误差分布文件。

### 7. MQTT 网关

在“系统设置”页面开启 MQTT 网关开关。

检查：

- Broker 地址和主题配置正确。
- 注册主题可创建设备。
- 上传主题可写入历史数据。

## 核心用例矩阵

| 编号 | 场景 | 输入或操作 | 预期结果 |
| --- | --- | --- | --- |
| T01 | 未登录访问管理接口 | 直接请求 `/api/dashboard` | 返回未登录或拒绝访问 |
| T02 | 用户登录 | 调用 `/api/auth/login` | Session 写入，`/api/auth/session` 返回用户信息 |
| T03 | 设备注册 | 调用 `/api/device/register` | 返回设备信息和 `api_key` |
| T04 | 密钥错误上报 | `X-API-Key` 错误 | 返回 `401`，历史数据不新增 |
| T05 | 正常读数上报 | 连续上传少于 24 条 | 历史数据新增，检测状态可为等待或跳过 |
| T06 | 窗口检测 | 同一设备累计 24 条读数 | 写入异常得分、阈值、预测标签和模型版本 |
| T07 | 低电压规则 | `battery_voltage` 明显偏低 | 告警生成，异常类型可解释 |
| T08 | 弱信号规则 | `signal_strength` 明显偏低 | 告警生成，严重程度正确 |
| T09 | 压力异常规则 | `pressure` 超出合理范围 | 告警生成，历史曲线可查看 |
| T10 | 告警反馈 | 调用 `/api/alerts/<id>/feedback` | 告警记录人工标注，训练样本可查询 |
| T11 | 工单闭环 | 创建、接单、完成工单 | 工单状态按流程变化，完成结果保留 |
| T12 | 重构曲线 | 请求 `/api/reconstruction` | 返回原始序列、重构序列和误差 |
| T13 | 持续训练 | 开启训练或调用 `/api/training` | 状态可查询，候选模型写入注册表 |
| T14 | 漂移监控 | 开启 `/api/drift` | 样本足够时返回漂移得分和特征变化 |
| T15 | WebSocket 刷新 | 页面保持打开并产生新告警 | 前端列表或统计刷新 |

## 答辩演示测试顺序

1. 启动后端和前端，确认登录页能打开。
2. 登录管理员账号，展示仪表盘、设备、告警、工单、历史数据和设置页。
3. 在设置页开启虚拟设备，等待历史数据增长。
4. 进入历史数据页，说明 24 点滑动窗口和 7 类特征。
5. 构造异常或等待模拟异常，展示告警生成。
6. 从告警进入工单闭环，完成接单和处理结果填写。
7. 展示 `/api/reconstruction` 对应的重构曲线，说明 LSTM AutoEncoder 的异常分数来源。
8. 展示训练、模型和漂移监控状态，说明后续维护能力。

## 回归检查清单

- 修改 `src/config.py` 后，检查 `start_system.bat` 默认值是否仍一致。
- 修改数据库模型后，检查仪表盘、历史数据、告警和工单页面。
- 修改检测逻辑后，检查 `/api/reconstruction` 和告警生成。
- 修改训练逻辑后，检查模型注册表和 active model 切换。
- 修改前端后，检查移动端宽度、表格显示和按钮反馈。

## 缺陷记录建议

测试发现问题时建议记录：

- 触发页面或接口。
- 请求参数或操作步骤。
- 实际响应、浏览器控制台错误或后端错误日志。
- 数据库中相关设备、读数、告警或工单 ID。
- 是否影响答辩演示主链路。
- 修复后使用哪个用例完成回归。

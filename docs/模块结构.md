# 模块结构

## 后端源码

```text
src/
  app.py                      Flask 应用、页面路由、API 路由、认证权限、WebSocket
  application/                应用编排层，承接后台服务生命周期和实时广播
  domain/                     业务域门面，提供设备、身份、监控、工单等稳定入口
  config.py                   路径、模型参数、数据库和环境变量配置
  database.py                 SQLAlchemy 模型和数据库初始化
  services.py                 核心业务服务和序列化逻辑
  detection_service.py        在线异常检测服务
  model.py                    LSTM AutoEncoder 模型定义
  preprocess.py               特征预处理
  model_registry.py           模型版本注册、激活和加载
  training_module.py          模型训练和持续训练入口
  training_data_generator.py  训练样本生成
  training_data_cleaner.py    训练数据清洗
  training_repository.py      训练数据访问
  training_store.py           训练状态存储
  drift_monitor.py            数据漂移监控
  device_integration.py       统一设备接入
  carrier_gateway.py          运营商 Webhook 数据适配
  mqtt_gateway.py             MQTT 主题订阅和消息处理
  simulation_module.py        虚拟燃气表数据生成
  security.py                 密码、密钥和安全工具
```

## 应用编排层

```text
src/application/
  runtime.py       管理模拟器、持续训练、MQTT 网关和漂移监控等长生命周期服务
  realtime.py      统一封装设备、工程师、用户、工单、告警和设置的 WebSocket 广播
```

该层不直接实现业务规则，也不定义数据库模型。它负责把路由触发的操作编排为后台服务启停、状态查询和实时事件推送，避免 `app.py` 继续累积跨模块胶水代码。

## 业务域门面

```text
src/domain/
  bootstrap.py     初始化相关门面
  devices.py       设备业务门面
  identity.py      用户和身份业务门面
  monitoring.py    监控和告警业务门面
  work_orders.py   工单业务门面
```

业务域门面当前对外暴露稳定函数名，底层实现仍复用 `services.py`、`device_integration.py` 等成熟模块。新增业务应优先先通过对应门面进入，再视复杂度拆出更细的领域服务。

## 前端资源

```text
frontend/
  index.html        独立前端入口
  package.json      前端依赖和启动脚本说明
  src/
    main.js         Vue 页面、状态、API、WebSocket、图表和地图逻辑
    styles.css      现代化前端样式
```

前端页面由独立 Vue 单页应用承载，不再使用 Flask 模板渲染。后端通过 `/api/*` 和 `/ws` 提供 JSON API 与实时推送。

## 脚本

```text
scripts/
  validate_project_health.py                         本地健康检查
  evaluate_lstm_autoencoder_experiment_results.py    LSTM AutoEncoder 实验评估导出
```

## 运行生成物

```text
data/
models/
instance/
```

这些目录用于保存本地数据、模型和数据库实例。生成物不作为系统源码或说明文档维护。

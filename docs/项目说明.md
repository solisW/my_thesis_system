# 项目说明

## 1. 课题名称

基于深度学习的物联网智能燃气表数据异常检测系统的设计与实现

## 2. 研究背景

随着物联网技术在智慧燃气领域的深入应用，智能燃气表已成为城市燃气信息化建设的重要基础设备。智能燃气表通过远程抄表、阀控管理、状态监测和异常预警等功能，提高了燃气企业运营效率。但在实际运行过程中，采集数据会受到设备老化、传感器故障、无线通信不稳定、环境扰动以及异常用气行为等因素影响，导致上报数据出现噪声、漂移、突变、缺失甚至伪造。

若不能及时识别这些异常数据，将直接影响计量准确性、用户安全管理和企业决策质量。因此，研究一种面向智能燃气表场景的高精度异常检测系统具有较强的工程意义和应用价值。

## 3. 系统目标

本系统旨在设计并实现一个基于深度学习的智能燃气表数据异常检测平台，完成以下目标：

- 构建符合燃气表业务特征的时序数据集
- 实现原始数据清洗、标准化和窗口化处理
- 构建 LSTM 自编码器异常检测模型
- 依据重构误差实现异常识别与阈值判定
- 提供简单直观的系统展示页面

## 4. 系统总体设计

系统总体由五部分组成：

1. 数据采集与模拟模块  
   模拟智能燃气表上传的瞬时流量、累计用气量、电池电压、信号强度、阀门状态、温度、压力等数据。

2. 数据预处理模块  
   对采集数据进行缺失值填充、异常值初步处理、标准化与滑动窗口构造。

3. 模型训练模块  
   使用正常样本训练 LSTM AutoEncoder，使模型学习正常时序模式。

4. 异常检测模块  
   对输入窗口进行重构，并以均方误差作为异常分数，若超过阈值则判定为异常。

5. 可视化展示模块  
   基于 Flask 搭建简易页面，实现检测结果预览和统计展示。

## 5. 核心算法原理

LSTM 自编码器由编码器和解码器组成。编码器将多维时间序列压缩为潜在表示，解码器再根据潜在表示重构原始序列。由于模型在训练阶段主要学习正常模式，因此对于异常数据窗口，模型难以准确重构，导致重构误差显著增大。系统据此实现异常识别。

异常分数定义如下：

```text
Score = mean((X - X_hat)^2)
```

其中，`X` 为原始输入窗口，`X_hat` 为模型重构结果。

## 6. 项目创新点

- 针对智能燃气表业务场景构造多维时序数据
- 综合考虑流量、电压、信号和压力等多源特征
- 使用深度学习方法替代传统静态阈值法
- 支持异常窗口级别检测与结果可视化

## 7. 测试方案

系统测试可从以下几个方面展开：

- 功能测试：验证数据生成、训练、推理和 Web 页面是否正常运行
- 性能测试：比较不同训练轮次下模型损失变化情况
- 检测效果测试：统计异常窗口识别率、误报率和召回率
- 鲁棒性测试：在缺失值、噪声扰动和不同异常强度下验证模型稳定性

## 8. 可扩展方向

- 接入真实燃气表数据和 MQTT/HTTP 实时上报链路
- 加入 Transformer、GRU、TCN 等模型进行对比实验
- 设计在线增量学习与动态阈值更新机制
- 增加用户登录、设备管理和历史告警分析模块

## 9. 结论

本项目实现了一个完整的智能燃气表异常检测原型系统，能够对多维时序数据进行自动建模和异常识别，具备较好的课程设计展示价值与后续扩展基础。

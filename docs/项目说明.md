# 项目说明

## 项目定位

本系统以论文初稿《基于深度学习的物联网智能燃气表数据异常检测系统的设计与实现》为工程基准，用于物联网智能燃气表运行数据的异常检测和运维管理。系统以 Flask 后端为核心，集成 PyTorch 版 LSTM AutoEncoder 模型，对燃气表多维时序数据进行重构误差检测，并把检测结果转化为告警、工单和人工反馈样本。

论文中的章节与工程实现对应关系如下：

| 论文内容 | 工程落点 |
| --- | --- |
| 数据接入与预处理模块 | `device_integration.py`、`carrier_gateway.py`、`mqtt_gateway.py`、`preprocess.py` |
| 深度学习异常检测引擎 | `detection_service.py`、`model.py`、`model_registry.py` |
| 告警与工单管理模块 | `services.py`、`src/domain/monitoring.py`、`src/domain/work_orders.py` |
| 实时监控与可视化看板 | `frontend/index.html`、`frontend/src/main.js`、`frontend/src/styles.css` |
| 持续训练流水线 | `training_module.py`、`training_data_cleaner.py`、`training_repository.py`、`drift_monitor.py` |

## 业务流程

```text
设备注册
  -> 设备上报读数
  -> 数据校验、缺失处理、字段映射与入库
  -> 按设备构造 24 点滑动窗口
  -> 标准化并输入 LSTM AutoEncoder
  -> 计算重构误差和异常得分
  -> 自适应阈值判定与业务规则复核
  -> 生成告警
  -> 派发工单
  -> 工程师处理并回写结果
  -> 反馈样本进入持续训练
```

## 核心模块

- 设备管理：维护燃气表编号、位置、通信密钥、启用状态和在线状态。
- 数据接入：统一处理 HTTP、MQTT、运营商 Webhook 和虚拟设备数据，接收论文定义的设备编号、时间戳、瞬时流量、累计用气量、电池电压、信号强度、阀门状态、环境温度和管网压力字段。
- 异常检测：构造 24 点窗口，使用 7 类模型特征调用 LSTM AutoEncoder 输出重构误差、异常得分、阈值和预测标签。
- 告警管理：保存异常类型、得分、阈值、模型版本和处理状态。
- 工单管理：支持派单、接单、处理中、完成和处理结果回写。
- 模型训练：生成训练样本，清洗数据，训练候选模型，评估后注册版本。
- 数据漂移监控：观察近期异常得分变化和模型性能衰减风险，必要时触发持续训练。
- 前端展示：提供仪表盘、设备、告警、工单、历史数据、报表、设置等页面。

## 技术栈

| 层次 | 技术 |
| --- | --- |
| 后端 | Python, Flask, Flask-SQLAlchemy |
| 数据库 | MySQL，支持 SQLite 本地测试 |
| 深度学习 | PyTorch, Scikit-learn, Joblib |
| 数据处理 | Pandas, NumPy |
| 实时通信 | Flask-Sock, WebSocket |
| 物联网接入 | HTTP/JSON, MQTT, Carrier Webhook |
| 前端 | Vue 3, ECharts, Leaflet, HTTP JSON API, WebSocket |

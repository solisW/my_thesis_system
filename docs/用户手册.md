# 用户手册

## 登录与首页

启动系统后访问：

```text
http://127.0.0.1:5173
```

用户登录后进入仪表盘。仪表盘展示设备总数、在线设备、异常设备、告警数量、工单状态和近期检测情况。

## 设备管理

设备页面用于维护智能燃气表基础信息：

- 设备编号。
- 设备名称。
- 安装位置。
- 所属片区。
- 通信协议。
- 启用状态。
- 在线状态。

新增设备后，系统会为设备生成或绑定通信密钥。真实设备和虚拟设备上报数据时都需要使用该密钥。

## 历史数据

历史数据页面展示燃气表上报记录，包括：

- 瞬时流量。
- 累计用气量。
- 电池电压。
- 信号强度。
- 阀门状态。
- 环境温度。
- 管网压力。
- 异常得分。
- 预测标签。
- 模型版本。

当某台设备累计数据达到 24 条后，系统可以构造完整滑动窗口并进行深度学习检测。

## 告警管理

告警页面展示系统识别出的异常事件。常见异常类型包括：

- 流量异常。
- 电压异常。
- 通信异常。
- 压力异常。
- 长时间流量静止。
- 综合异常。

告警包含异常得分、模型阈值、严重程度、处理状态和模型版本。管理员可根据告警创建工单，也可以对告警进行人工反馈。

## 工单管理

工单用于把异常事件转化为运维任务。

典型流程：

1. 管理员查看告警。
2. 管理员创建工单并选择工程师。
3. 工程师接单。
4. 工程师处理现场问题。
5. 工程师填写处理结果并完成工单。
6. 系统保存处理记录，用于后续模型优化。

## 工程师管理

工程师页面用于维护运维人员信息和在线状态。派单时系统可根据工程师状态和区域信息提供候选人。

管理员和工程师的姓名、工号、电话、片区和地址等身份信息在数据库中以加密字段保存，页面展示时由后端解密后返回当前登录会话。

## 报表与重构曲线

报表页面用于查看系统统计结果。重构曲线用于解释模型检测结果：

- 原始曲线：设备上传的真实时序数据。
- 重构曲线：LSTM AutoEncoder 根据正常模式恢复出的序列。
- 误差曲线：原始数据和重构数据之间的差异。

误差越高，说明当前窗口越偏离正常运行模式。

## 系统设置

设置页用于查看：

- 当前模型版本。
- 训练状态。
- 模型阈值。
- MQTT 网关状态。
- 虚拟设备模拟器状态。
- 数据漂移监控状态。

管理员可根据需要启动训练、控制模拟器或查看系统运行状态。

## 答辩演示前检查

演示前建议先运行 `.\start_system.bat`，确认浏览器能打开 `http://127.0.0.1:5173`，并检查登录页、综合态势和系统设置页面正常后再进入演示。

# 前后端分离说明

系统已新增独立前端工程，前端不再依赖 Flask 模板渲染。

## 技术栈

- 前端：Vue 3、ECharts、Leaflet
- 后端：Flask、Flask-SQLAlchemy、Flask-Sock
- 通信：HTTP JSON API、WebSocket
- 后端分层：`app.py` 作为接入层，`src/application/` 负责后台服务和实时广播编排，`src/domain/` 作为业务域入口

## 目录

```text
frontend/index.html      独立前端入口
frontend/src/config.js   前端运行配置、页面元数据和地图参数
frontend/src/api.js      HTTP JSON API 客户端，统一处理 Cookie 会话和错误消息
frontend/src/formatters.js 通用默认状态、表单默认值和显示格式化函数
frontend/src/map-utils.js 地图坐标转换与点位解析
frontend/src/main.js     Vue 页面状态、WebSocket、ECharts、Leaflet 渲染编排
frontend/src/styles.css  前端样式
frontend/package.json    前端工程描述
```

迁移前的 `frontend/templates/` 和 `frontend/static/` 已删除。当前前端只维护独立 Vue 入口和 `frontend/src/` 资源，后端页面路由仅负责重定向到独立前端。

## 启动

完整系统：

```powershell
.\start_system.bat
```

该脚本会同时启动后端 API 和独立前端。根目录不再保留单独前端启动脚本，演示和日常使用统一运行 `start_system.bat`。

访问：

```text
前端：http://127.0.0.1:5173
后端：http://127.0.0.1:5000
```

系统启动后直接访问前端页面检查登录页、综合态势和系统设置等核心功能。

## 接口边界

- 登录：`POST /api/auth/login`
- 会话：`GET /api/auth/session`
- 退出：`POST /api/auth/logout`
- 业务数据：`/api/*`
- 实时推送：`/ws`

前端请求默认携带 Cookie 会话，后端通过 `FRONTEND_ORIGINS` 允许独立前端来源跨域访问。

## 后端分层补充

`src/app.py` 继续作为 Flask 应用工厂和路由注册入口。横切逻辑已下沉到应用层：

- `src/application/auth.py`：当前用户、登录校验、角色校验和页面上下文。
- `src/application/http.py`：独立前端跨域访问配置。
- `src/application/realtime.py`：WebSocket 广播编排。
- `src/application/runtime.py`：模拟器、训练、MQTT、漂移监控等后台服务生命周期。

业务规则仍集中在 `src/domain/`，数据库模型和外部协议适配仍由 `src/database.py`、`src/*_gateway.py` 等基础设施模块承载。

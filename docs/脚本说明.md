# 脚本说明

根目录保留后端启动、前端启动、健康检查和实验导出脚本。模拟设备、持续训练、数据漂移监控和 MQTT 网关已经接入 Web 系统，在“系统设置”页面通过开关按钮启停，不再作为独立启动脚本维护。

## 启动脚本

| 脚本 | 入口 | 用途 |
| --- | --- | --- |
| `start_system.bat` | `python -m flask --app src.app run` 与 `python -m http.server 5173` | 启动完整演示系统，包括后端 API、WebSocket、模块控制入口和独立前端 |
| `start_frontend.bat` | `python -m http.server 5173` | 单独启动独立前端静态服务 |
| `validate_project_health.bat` | `python scripts/validate_project_health.py` | 执行本地健康检查 |
| `system_health_check.bat` | `python scripts/system_health_check.py` | 在系统运行后检查前端资源、后端接口、登录和看板 |
| `export_lstm_autoencoder_experiment_results.bat` | `python scripts/evaluate_lstm_autoencoder_experiment_results.py` | 导出 LSTM AutoEncoder 实验评估结果 |

## 推荐启动流程

1. 启动 MySQL，并确认 `gas_monitor` 数据库存在。
2. 运行 `start_system.bat`，该脚本会同时拉起后端和前端窗口。
3. 打开 `http://127.0.0.1:5173`。
4. 登录后进入“系统设置”，按需开启模拟设备、持续训练、漂移监控或 MQTT 网关。
5. 系统运行后可执行 `system_health_check.bat` 做完整运行态诊断。

## 健康检查

```powershell
.\validate_project_health.bat
```

健康检查覆盖 Python 编译、前端脚本语法、Flask 应用创建和关键业务入口冒烟测试。

系统运行态诊断：

```powershell
.\system_health_check.bat
```

该诊断需要前端和后端已经启动，会检查前端静态资源、后端根接口、登录接口和看板接口。

## 脚本使用注意

- `start_system.bat` 默认使用 MySQL，并补齐后端、前端、异步检测、漂移监控和 MQTT 的默认环境变量；如果前后端端口已被占用，会复用已运行服务。
- `validate_project_health.bat` 默认使用 SQLite，并关闭漂移监控，避免本地快速检查依赖 MySQL 和后台定时任务。
- `start_frontend.bat` 只托管 `frontend/` 静态文件，不会启动后端 API。
- `system_health_check.bat` 依赖已启动服务，适合答辩演示前最后确认。
- `export_lstm_autoencoder_experiment_results.bat` 会向 `data/experiment_results/` 写入实验评估结果，该目录属于运行生成物。

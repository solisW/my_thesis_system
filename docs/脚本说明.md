# 脚本说明

根目录只保留一个批处理入口：

```powershell
.\start_system.bat
```

提交或答辩前的本地验收使用 Python 自检脚本：

```powershell
python scripts\validate_project_health.py
```

该脚本不会启动浏览器窗口，默认切换到 SQLite 验证环境，并检查前端静态资源、管理员登录、核心 API、设备注册/上报、运营商 Webhook、训练、漂移监控和 MQTT 状态接口。

该脚本会同时启动：

- 后端 API 与 WebSocket 服务：`http://127.0.0.1:5000`
- 独立前端静态服务：`http://127.0.0.1:5173`

## 启动流程

1. 启动 MySQL，并确认 `gas_monitor` 数据库存在。
2. 在项目根目录运行 `.\start_system.bat`。
3. 脚本会检查依赖和端口，并在当前窗口内同时启动后端和前端。
4. 服务就绪后，手动在浏览器打开 `http://127.0.0.1:5173`。
5. 登录后进入“系统设置”，按需开启模拟设备、持续训练、漂移监控或 MQTT 网关。

## 注意事项

- `start_system.bat` 默认使用 MySQL。
- 如果前端端口 `5173` 或后端端口 `5000` 已被占用，脚本会提示先关闭旧服务；这样才能保证本次启动的服务可由一次 `Ctrl+C` 统一关闭。
- 使用系统期间请保持当前启动窗口打开；需要停止系统时，在该窗口按一次 `Ctrl+C`。
- 不再保留单独前端启动、健康检查和实验导出批处理脚本；相关 Python 工具仍在 `scripts/` 目录，可由开发者按需手动运行。
